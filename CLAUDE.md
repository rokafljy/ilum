## 🎯 프로젝트 방향 (2026-07-07 합의)

- **v2 재구축** (기존 코드 개조 아님). 기존 도메인 지식·UI 패턴은 자산으로 이식.
- 스택 방향: Supabase Postgres **정규화 스키마 + RLS 테넌트 격리** / Supabase Auth / React 모듈 분리 / Cloudflare 배포.
- 기존 시스템의 JSONB 단일 row + `_u` 병합 방식은 **가져가지 않는다** (row 단위 업데이트로 대체).
- 사용자 역할: 슈퍼관리자(운영사=우리) / 기관 관리자 / 기관 담당자 / 멘토 / 청년(팀).
- 단계: Phase 0 시장검증 → 1 기반구축 → 2 파일럿(건국대 마이그레이션 + 외부 2~3기관) → 3 확장.
- 규제 유의: 청년 개인정보·통장내역 = 수탁 데이터. 데이터 사업은 익명화·가명처리 기반 부가서비스 형태.
